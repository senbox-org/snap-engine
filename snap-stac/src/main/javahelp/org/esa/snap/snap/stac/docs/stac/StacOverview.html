SpatioTemporal Asset Catalog (STAC)

STAC is an open specification that provides a common language for describing geospatial information so it can be more easily worked across providers and tools. The Sentinel Toolbox can open a STAC Item's referenced assets as a single product, and can also write a product out as a STAC Item with its rasters saved as Cloud Optimized GeoTIFFs.

A STAC Catalog is a hierarchy of three kinds of entities:

  • Catalog - a top-level object that groups Collections and Items together.
  • Collection - a group of related Items (for example, all scenes acquired by one sensor over one area).
  • Item - a single footprint and time stamp pairing one or more raster assets (e.g. GeoTIFFs) with a small JSON sidecar of metadata. A STAC Item is the basic unit the Toolbox reads and writes.

The STAC specification is extensible. The Toolbox understands several standard extensions when they are present in the Item's JSON, including eo, proj, sar, sat, view, raster, and the snap extension used for round-tripping SNAP-specific metadata through the STAC writer.

Using STAC in the Toolbox

  • Import STAC - open a STAC Item JSON file as a product.
  • Export STAC - save the current product as a STAC Item with COG assets.
  • Using the StacClient - programmatic API to browse a remote STAC catalog, search for Items and download their assets.
