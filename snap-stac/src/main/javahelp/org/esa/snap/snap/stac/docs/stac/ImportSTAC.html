Import STAC

This command opens a STAC Item as a product. The input is the Item's JSON sidecar; the Toolbox loads the rasters referenced from its assets block (typically GeoTIFF / COG files) and merges STAC metadata into the product's abstracted metadata.

How to open a STAC Item

  1. Choose File → Import → Generic Formats → STAC, or drag the Item JSON into the Product Explorer.
  2. Select the STAC Item JSON file (file extension .json).

The Toolbox recognises a JSON file as a STAC Item when its top-level object declares "type": "Feature" and contains stac_version plus an assets block. Files that do not match are passed on to other readers.

What is loaded

  • Every raster asset referenced in the Item (local path or HTTP/HTTPS URL) becomes a band in the target product. Non-raster assets (thumbnails, XML sidecars, ...) are skipped.
  • Asset roles, EO band identifiers and wavelengths from the eo:bands extension are used to name bands and populate band metadata where available.
  • Geo-coding is taken from the proj extension when present (EPSG code, WKT, or the transform / shape pair). Otherwise the geographic footprint from geometry / bbox is used as a fallback.
  • SAR metadata from the sar extension (polarisations, product type, instrument mode) and platform / instrument identifiers are mapped into the product's abstracted metadata so the rest of the Toolbox can consume them.

Remote assets

Asset href values may be absolute URLs (http://, https://). The reader will stream those assets through GDAL's virtual file system if the environment can reach them. Authentication and signed-URL support are delegated to GDAL - configure the usual environment variables (AWS_*, GDAL_HTTP_*, ...) before launching SNAP if the catalog requires them.

Constraints

  • Only STAC Items are opened directly. To browse a STAC Catalog or Collection, pick an Item from it first.
  • Assets with different resolutions are preserved as multi-size bands. Resample the product before running operators that require a single raster grid.
  • STAC extensions that the reader does not understand are preserved verbatim in the product's Original Metadata but are not otherwise interpreted.
