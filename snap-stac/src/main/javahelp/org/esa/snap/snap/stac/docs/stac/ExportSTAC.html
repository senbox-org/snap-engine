Export STAC

Export STAC Product

This writes the current product as a STAC Item. Two kinds of files are produced side by side:

  • An Item JSON sidecar (<productName>.json) holding the Item metadata, extension fields and the assets block.
  • One Cloud Optimized GeoTIFF (.tif) per band (or per group of bands), referenced from the Item's assets.

How to export

  1. With a product selected, choose File → Save Product As... (or File → Export Product when running from a graph) and pick the STAC format.
  2. Select a target folder. The writer places the JSON sidecar and the GeoTIFF assets in that folder.

The writer is also available from GPF graphs and the gpt command line through the -f STAC format selector.

What is written

  • Geographic footprint as the Item's geometry / bbox.
  • Projection details (EPSG code when known, WKT otherwise, plus the affine transform and shape) under the proj extension.
  • Band-level information (units, data type, no-data, scaling) under the raster extension, and wavelengths under eo:bands when available.
  • Platform / instrument / SAR product fields from the product's abstracted metadata under the sat, instrument and sar extensions.
  • SNAP-specific fields (product type, SNAP version, band unit strings that do not have a direct STAC equivalent) under the snap extension so that a round-trip through the STAC reader preserves them.

Constraints

  • Multi-size products cannot be written as a single Item. Resample the product to a single grid before export.
  • Products without a CrsGeoCoding are written with only the geographic footprint; the proj extension will be incomplete and some downstream STAC tooling may reject the Item. Run a geocoding or reprojection step first for best interoperability.
  • Each output band is written as a separate Cloud Optimized GeoTIFF. Virtual bands are materialised into regular rasters at write time.
  • The writer produces a single Item; generating a Collection or Catalog that groups multiple Items is outside the scope of this operator and should be done with external STAC tooling.
