Using the StacClient

Using the StacClient

StacClient is a small Java API in the snap-stac module for talking to a remote STAC service from inside the Toolbox or from your own code. It wraps the three operations you typically need: inspect a Catalog's Collections, search for Items, and download (or stream) an Item's assets. It is a pure-code entry point - there is no dialog for it in SNAP Desktop - so use it from the Tools → Plugins → Scripting console, a Groovy/Jython script, or a plug-in that depends on snap-stac.

Creating a client

Construct a client with the root URL of a STAC API:

import org.esa.snap.stac.StacClient;
import org.esa.snap.stac.StacCatalog;
import org.esa.snap.stac.StacItem;

StacClient client = new StacClient("https://planetarycomputer.microsoft.com/api/stac/v1");

When the URL points at Microsoft's Planetary Computer the client auto-installs a signing modifier so that asset downloads get a short-lived signed URL. For other catalogs that require signing, pass a DownloadModifier explicitly: 

import org.esa.snap.stac.internal.DownloadModifier;
import org.esa.snap.stac.internal.EstablishedModifiers;

// built-in Planetary Computer modifier (equivalent to the auto default)
StacClient pc = new StacClient(
        "https://planetarycomputer.microsoft.com/api/stac/v1",
        EstablishedModifiers.planetaryComputer());

// custom modifier - rewrite each href before download
DownloadModifier mine = input -> input + "?token=" + token;
StacClient custom = new StacClient("https://my.catalog/stac/v1", mine);

If you already have a StacItem and want a client pointing at the Catalog it belongs to, use the static factory: 

StacItem item = new StacItem("https://.../collections/landsat-c2-l2/items/LC09_...");
StacClient client = StacClient.createClient(item);   // follows the "root" link

Inspecting the Catalog

StacCatalog catalog = client.getCatalog();

System.out.println("Catalog id      : " + catalog.getId());
System.out.println("STAC version    : " + catalog.getVersion());
System.out.println("Num collections : " + catalog.getNumCollections());

for (String name : catalog.listCollections()) {
    System.out.println("  " + name);
}

// drill into one Collection
StacCollection col = catalog.getCollection("sentinel-2-l2a");

Searching for Items

Three overloads cover the common cases. All return a StacItem[]; the client follows rel="next" links transparently, so results from multi-page queries come back merged.

By bounding box and optional datetime range

// west, south, east, north (EPSG:4326, decimal degrees)
double[] bbox = {-124.2751, 45.5469, -123.9613, 45.7458};

// null datetime means "any time"; otherwise pass "start/end" or a single ISO-8601 instant
StacItem[] items = client.search(
        new String[]{"sentinel-2-l2a", "landsat-c2-l2"},
        bbox,
        "2022-05-01T00:00:00Z/2022-05-31T23:59:59Z");

By arbitrary GeoJSON geometry

import org.json.simple.JSONObject;
import org.json.simple.parser.JSONParser;

JSONObject polygon = (JSONObject) new JSONParser().parse(
        "{ \"type\": \"Polygon\", \"coordinates\": [[[-124,45],[-123,45],[-123,46],[-124,46],[-124,45]]] }");

StacItem[] items = client.search(
        new String[]{"sentinel-2-l2a"},
        polygon,
        null);

With arbitrary query parameters - useful for catalog-specific filters:

import java.util.HashMap;
import java.util.Map;

Map<String, Object> params = new HashMap<>();
params.put("bbox",     "20.22,43.69,29.63,48.22");
params.put("datetime", "2022-05-01T00:00:00Z/2022-05-02T23:59:59Z");
params.put("query",    "{\"eo:cloud_cover\":{\"lt\":10}}");   // URL-encode as needed

StacItem[] items = client.search(new String[]{"sentinel-2-l2a"}, params);

// paginated variant: page index (1-based) and page size
StacItem[] firstPage = client.search(
        new String[]{"sentinel-2-l2a"}, params, 1, 50);

Collection names that are not present in the catalog are silently filtered out of the request. If none of the requested collections exist, search returns an empty array.

Walking an Item's assets

import org.esa.snap.stac.extensions.Assets;

for (StacItem item : items) {
    System.out.println(item.getId() + "  " + item.getDateTime());

    for (String assetId : item.listAssetIds()) {
        Assets.Asset a = item.getAsset(assetId);
        System.out.println("  " + assetId + "  " + a.getURL());
    }
}

Downloading assets

downloadAsset writes one asset to disk; downloadItem creates a sub-folder named after the Item's id and writes every asset into it. Both honour the client's DownloadModifier, so URL signing happens transparently. 

import java.io.File;

File workDir = new File("C:/temp/stac-out");

// single asset
Assets.Asset asset = items[0].getAsset(items[0].listAssetIds()[0]);
File savedFile = client.downloadAsset(asset, workDir);

// entire item (one sub-folder per Item, one file per asset)
File itemFolder = client.downloadItem(items[0], workDir);

For very large rasters, prefer streaming over downloading the whole file. streamAsset opens an InputStream at the asset's (signed) URL; pair it with a GDAL VSI path or /vsicurl/ reader if you want random access without first saving to disk. 

import java.io.InputStream;

try (InputStream in = client.streamAsset(asset)) {
    // consume, or hand off to a streaming parser
}

// just resolve the signed URL, without opening a stream:
String signed = client.signURL(asset);

Opening a downloaded Item in SNAP

The files dropped by downloadItem include the Item's own JSON sidecar plus its raster assets. To open that folder as a SNAP product, point the STAC reader at the JSON file. When using SNAP's ProductIO directly from the same JVM: 

import org.esa.snap.core.dataio.ProductIO;
import org.esa.snap.core.datamodel.Product;

File jsonSidecar = new File(itemFolder, itemFolder.getName() + ".json");
Product product = ProductIO.readProduct(jsonSidecar);

Tips and gotchas

  • Signing via EstablishedModifiers.planetaryComputer() does a synchronous HTTP round-trip per signed URL with exponential backoff on HTTP 429. Sign once, cache the result for the lifetime of the download.
  • The search overloads that take an explicit String for the datetime forward it into the STAC query without URL-encoding. Pre-encode values that contain characters outside A-Za-z0-9_.:/- if your HTTP stack is strict.
  • Asset URLs may carry query strings (signed parameters, cache-busters). downloadAsset strips any ?... suffix from the output filename, so you get clean names on disk.
  • Pagination in the search(..., int pageNumber, int pageSize) overload kicks in only when both arguments are positive.
